Hora da prática – Próxima Atividade

Após entendermos como configurar subredes e adotar o conceito de VLANs para segmentar e robustecer a rede, avançaremos explorando aspectos para tornar o projeto mais eficiente.

É importante que tenha feito a lista de atividades da aula anterior, pois usaremos aquele projeto como base.

Atividades
Derrubar um link entre switches
No projeto com switches interligando três setores da empresa, remova um cabo de conexão entre dois switches. Observe o que ocorre.

Inserir e configurar um novo switch para redundância
Adicione um novo switch, conecte-o aos switches existentes para criar redundância nas conexões, tornando a rede menos vulnerável a quedas.

Testar operação da rede com redundância
Desconecte um link fisicamente e, no modo simulação do Packet Tracer, observe a atuação dos protocolos, como STP, garantindo a continuidade da comunicação.

Calcular endereçamento IP para 450 dispositivos por setor

Converta 450 decimal para binário (450)10 = (111000010)2 → 9 bits necessários

Ajuste a máscara para atender a essa demanda usando endereços classe B

Máscara padrão 255.255.0.0 → binário 11111111.11111111.00000000.00000000

Máscara ajustada 255.255.254.0 → binário 11111111.11111111.11111110.00000000

Divida o espaço em sub-redes, por exemplo

Sub-rede 1 172.16.0.0 — Broadcast 172.16.1.255

Sub-rede 2 172.16.2.0 — Broadcast 172.16.3.255

Sub-rede 3 172.16.4.0 — Broadcast 172.16.5.255

Implementar máscara e sub-redes no projeto
Atualize as configurações dos dispositivos (switches, roteadores) no Packet Tracer para refletir a nova máscara de rede e as sub-redes calculadas, garantindo consistência.
